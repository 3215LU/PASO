## 1. Declaración del Proyecto (Project Statement)

Este proyecto tiene como objetivo desarrollar e implementar un sistema informático integral para la gestión de la elección de "Chico 10", "Mis Paisana" y "Embajadora de los Estudiantes". El sistema optimizará el proceso de evaluación y votación, asegurando transparencia, eficiencia y facilidad de uso para los jurados. Además, proporcionará datos y reportes valiosos para la institución.

## 2. Propósito del Proyecto (Project Purpose)

*   Automatizar el proceso de votación y evaluación para reducir el tiempo y los recursos necesarios.
*   Aumentar la transparencia y la confianza en el proceso electoral.
*   Mejorar la experiencia de los jurados mediante una plataforma intuitiva y fácil de usar.
*   Generar informes detallados y métricas que permitan el análisis de los resultados y la toma de decisiones.
*   Modernizar la imagen de la institución al adoptar tecnología de vanguardia en sus procesos.

## 3. Justificación del Proyecto (Project Justification)

El sistema actual se basa en métodos manuales, que son propensos a errores, requieren mucho tiempo y son difíciles de auditar. La implementación de este sistema informático permitirá:

*   **Reducir costos:** Eliminar la necesidad de imprimir boletas, tabular resultados manualmente y realizar otras tareas administrativas.
*   **Mejorar la precisión:** Eliminar errores humanos en el cálculo de los puntajes y la generación de informes.
*   **Aumentar la eficiencia:** Acelerar el proceso de votación y evaluación, liberando tiempo para otras actividades importantes.
*   **Fortalecer la transparencia:** Proporcionar un registro claro y auditable de todas las votaciones, lo que aumenta la confianza en los resultados.
*   **Impulsar la innovación:** Demostrar el compromiso de la institución con la adopción de tecnologías modernas.

## 5. Alcance del Proyecto (Project Scope)

*   **Incluido:**
    *   Desarrollo de una aplicación web para la votación y evaluación.
    *   Desarrollo de una base de datos para almacenar información de candidatos, jurados y resultados.
    *   Implementación de medidas de seguridad para proteger los datos.
    *   Capacitación de los jurados en el uso del sistema.
    *   Generación de informes y estadísticas.
    *   Soporte técnico durante el período de garantía.
*   **Excluido:**
    *   Integración con sistemas existentes (a menos que se especifique lo contrario en una fase posterior).
    *   Desarrollo de aplicaciones móviles nativas.
    *   Mantenimiento continuo después del período de garantía (se puede contratar por separado).
    *   Personalización extensiva del sistema más allá de las funcionalidades definidas.

## 6. Requisitos de Alto Nivel (High-Level Requirements)

*   El sistema debe permitir la configuración de categorías de evaluación personalizables.
*   El sistema debe permitir la carga de información y fotos de las candidatas.
*   El sistema debe permitir a los jurados registrar sus puntuaciones por cada categoría.
*   El sistema debe calcular automáticamente el puntaje total de cada candidata.
*   El sistema debe mostrar un ranking de las candidatas en tiempo real.
*   El sistema debe operar en una red local y estar accesible a través de internet con las debidas medidas de seguridad (autenticación, encriptación).
*   El sistema debe contar con una interfaz de usuario intuitiva y fácil de usar para los jurados.
*   El sistema debe generar reportes y estadísticas de la votación en formatos PDF y Excel.
*   El sistema debe ser compatible con los navegadores web más comunes (Chrome, Firefox, Safari).
*   El sistema debe cumplir con las leyes y regulaciones de protección de datos aplicables.

* **Director/a de la Institución:** Aprobación del proyecto, asignación de recursos. Expectativa: Sistema funcional y fácil de usar.  
* **Secretaría Académica:** Gestión de datos de candidatos y jurados, soporte en la capacitación. Expectativa: Reducción de la carga de trabajo manual.  
* **Jurados:** Uso del sistema para la votación y evaluación. Expectativa: Interfaz intuitiva y proceso de votación eficiente.  
* **Estudiantes:** Usuarios finales indirectos, beneficiarios del proceso transparente. Expectativa: Proceso de elección justo y transparente.  
* **Equipo de Desarrollo:** Desarrollo e implementación del sistema. Expectativa: Claridad en los requisitos y alcance del proyecto.

* **Impacto:** Incumplimiento de los plazos del proyecto, posible insatisfacción de los interesados.  
* **Mitigación:** Planificación detallada, gestión proactiva del cronograma, asignación de recursos adicionales.  
* **Riesgo:** Falta de adopción del sistema por parte de los usuarios (jurados).  
* **Impacto:** Subutilización del sistema, necesidad de continuar con procesos manuales.  
* **Mitigación:** Diseño centrado en el usuario, capacitación adecuada, soporte técnico continuo.  
* **Riesgo:** Vulnerabilidades de seguridad que comprometan la integridad de los datos.  
* **Impacto:** Pérdida de confianza en el sistema, posibles fraudes en la votación.
